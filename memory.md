This project is used to track people profiles, project updates, and lightweight management context.

- `Projects/` - project-specific working context.
- `Trackers/` - recurring tracking artifacts.

## Update Rules

When creating a project update:

- Read `updates.md` first.
- Keep it single-screen unless detail is requested.
- Lead with the conclusion.
- Use a compact milestone table.
- Keep risks/decisions to the top 3-5.
- Link only the strongest evidence.
- Mark missing ETA, owner, cost, or metrics as manual fill.
- Do not invent data.

When creating an update for a specific person:

- Check `Profiles/` first.
- Tune the update to that person's style.

## Ramki

Ramki prefers concise, decision-oriented visibility.

For Ramki updates:

- Use RAG / ETA / status / ask.
- Keep evidence links minimal.
- Ask him only for decisions, escalation, or sponsorship.
- Avoid long background and ticket dumps.
