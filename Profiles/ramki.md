# Ramki

Last updated: 2026-05-22

## Style

Ramki needs fast visibility.

Use:

- one screen,
- RAG,
- ETA,
- blocker,
- owner,
- ask only if needed.

## Cares About

- Is it on track?
- What changed?
- What is blocked?
- Customer / bank impact.
- Launch risk.
- External dependency.
- Spend signal, if real.

## Good Update

```text
Status: Red / Amber / Green
Summary: <1-2 lines>

| Milestone | RAG | ETA | Status | Ask |

Top risks:
1. <risk> - <proof> - <decision if needed>
```

## Avoid

- long background,
- ticket dump,
- too many asks,
- fake cost,
- fake ETA,
- asking Ramki for things Deepak can decide.

## Signal

EON update proved this: short wins. One-screen wins.
