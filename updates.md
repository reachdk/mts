# Update Rules

Use this for project updates.

## Shape

- One screen.
- Start with status.
- Use one small table.
- Show top 3-5 risks or choices.
- Link only key proof.
- Say `manual fill` when data is missing.

## Style

- Short.
- Clear.
- Decision first.
- No long story.
- No ticket dump.
- No fake dates.
- No fake cost.
- No ask unless the person can help.

## Proof

- Main Jira filter.
- Key blocker tickets.
- Key Confluence page.
